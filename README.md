# Сервис регистрации

Проект реализует базовый сервис регистрации пользователей на C++ с валидацией входных данных, проверкой конфликтов по email и username, ограничением по роли и обработкой регистрационных запросов.

## Что было реализовано

В рамках проекта были выполнены следующие шаги:

- создана структура приложения с отдельными слоями для валидации, бизнес-логики, HTTP-обработки и хранения данных;
- реализована проверка обязательных полей, корректности email, допустимости роли и ограничения размера метаданных;
- добавлен механизм обработки регистрационных сценариев с возвратом понятных сообщений для успешной регистрации и ошибок;
- реализован парсинг JSON-запросов, обработка HTTP-запросов и запуск простого HTTP-сервера;
- добавлено хеширование паролей и генерация идентификаторов пользователей;
- создан набор тестов, который проверяет успешную регистрацию, дублирование email, дублирование username, запрещённую роль и слишком большие метаданные;
- настроена сборка через CMake и подготовлен проект для запуска и тестирования в локальной среде.

## Возможности

- Проверка обязательных полей и формата email
- Запрет регистрации с ролью admin
- Ограничение размера метаданных
- Проверка дубликатов по email и username
- Обработка JSON-запросов и HTTP-ответов
- Хеширование паролей и генерация идентификаторов пользователей

## Структура проекта

- src/registration_logic.{hpp,cpp} — правила валидации
- src/registration_service.{hpp,cpp} — бизнес-логика регистрации
- src/http_handler.{hpp,cpp} — обработка входящих запросов
- src/http_server.{hpp,cpp} — запуск HTTP-сервера
- src/json_parser.{hpp,cpp} — разбор JSON-тела запроса
- src/password_hasher.{hpp,cpp} — хеширование паролей
- src/user_repository.{hpp,cpp} — хранение пользователей
- src/uuid_generator.{hpp,cpp} — генерация идентификаторов
- tests/test_validation.cpp — набор проверок для основных сценариев

## Как это работает

1. Входящий запрос поступает в обработчик HTTP.
2. Тело запроса разбирается из JSON.
3. Данные проходят валидацию.
4. Сервис проверяет конфликты по email и username.
5. При успехе пользователь сохраняется и возвращается результат регистрации.
6. При ошибке возвращается код и сообщение, понятное для клиента.

## Сборка и запуск

Откройте PowerShell и выполните:

```powershell
cd C:\task
cmake -S . -B build
cmake --build build --config Debug
```

Запуск тестов:

```powershell
cd C:\task\build
.\auth_service_tests.exe
```

Запуск сервиса:

```powershell
cd C:\task\build
.\auth_service.exe
```
